# AGENTS.md — Laju Go Project Conventions

## Summary

AGENTS.md is the canonical agent instruction file for the **Laju Go** project — a high-performance SaaS boilerplate built with Go Fiber + Svelte 5 + Inertia.js + SQLite + templ. It defines the project's strict architectural layering (Handler → Service → Query), code generation workflow (sqlc for queries, templ for templates), and critical conventions every agent must follow.

The document covers the full development lifecycle: commands (npm/Make), migration conventions (one table per file), Inertia.js patterns (X-Inertia headers, `use:inertia` for SPA nav, `router.post()` for forms), Svelte 5 rules (`$effect` only for external side effects, not derived state), CSRF handling (manual `fetch()` requires `X-XSRF-TOKEN` header), templ gotchas (SVG escaping via `@templ.Raw`), design standards (visual hierarchy, whitespace, 60-30-10 color), testing strategy (Go unit + agent_browser E2E), and deployment (Git-based or Docker).

Several pink/red-flagged rules are non-negotiable: handlers must never call queries directly, internal links must use `use:inertia`, and manual `fetch()` requests to CSRF-protected routes must include the `X-XSRF-TOKEN` header.

## Key Takeaways

- **Architecture layering is strictly enforced**: Handler → Service → Query. Handlers must never call queries directly; tests are the only exception for data setup.
- **Code generation drives two paths**: sqlc generates `app/queries/` from `queries/*.sql` (never edit manually), and `templ generate` compiles `.templ` files to `*_templ.go`.
- **Migration convention — one table per file**: Each migration file contains exactly one `CREATE TABLE` statement with `-- +goose Up` and `-- +goose Down` sections.
- **Svelte 5 `$effect` anti-pattern**: Never use `$effect` for derived state or prop initialization — use `$derived()` or direct `$state(initial)` instead.
- **CSRF for manual `fetch()`**: All manual `fetch()` calls to `/app/*` or `/admin/*` must include the `X-XSRF-TOKEN` header from `getCSRFToken()` helper.
- **Inertia SPA navigation**: Internal links must use `use:inertia` action; OAuth links use plain `<a>` tags; form submissions use `router.post/put/delete`.
- **Design standards**: Mobile-first responsive design with 60-30-10 color rule, 16-24px minimum padding, and micro-interactions on all interactive elements.
- **E2E testing via agent_browser**: No Playwright/Cypress — real browser testing through pi's agent_browser, including session injection for protected pages.
