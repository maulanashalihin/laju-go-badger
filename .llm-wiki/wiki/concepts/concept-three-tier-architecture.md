# Three-Tier Architecture (Handler → Service → Query)

Laju Go enforces a strict three-tier architecture: **Handler → Service → Query**.

## The Rule

Handlers must **never** call queries directly. All database access must go through services. This is the most important rule in the project.

```
routes/web.go → app/handlers/ → app/services/ → app/queries/ → SQLite
```

## Layer Responsibilities

- **Handlers**: Parse requests, call services, return responses. Zero business logic.
- **Services**: Business logic, auth flows, external API integration.
- **Queries**: Generated by sqlc from `queries/*.sql`. Never edited manually.

## Exception

Test files (`*_test.go`) may call queries directly for test data setup.
